# 远程仓库
Git是分布式版本控制系统，同一个Git仓库，可以分布到不同的机器上。怎么分布呢？
最早，肯定只有一台机器有一个原始版本库，此后，别的机器可以“克隆”这个原始版本库，而且每台机器的版本库其实都是一样的，并没有主次之分。

Git的杀手级功能之一（注意是之一，也就是后面还有之二，之三……）：远程仓库。

## 添加远程仓库
本地提交以后，推送远程仓库
```
git push origin master
```

## 从远程仓库克隆
要克隆一个仓库，首先必须知道仓库的地址，然后使用`git clone`命令克隆。
Git支持多种协议，包括`https`，但`ssh`协议速度最快。
```
git clone <repository URL>
```

## 其他仓库操作
Github star什么的这些都比较简单，都有对应的按钮。
删除一个自己Github的仓库，这要到你对应的Responsity的设置（setting）里面，找到删除按钮，确认删除。

fork操作，这个就是把别人的仓库克隆一份到你自己的哪里，你就有一个一样的副本仓库。
fork和clone不一样，clone相当于大家用同一个仓库，大家一起开发。每一次fork，就创建一个对应仓库的副本，你在这个仓库副本仓库操作对之前的仓库没有影响，彼此是两个独立的仓库。
